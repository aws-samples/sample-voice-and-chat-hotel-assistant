# Improvements

## Development and deployment

- **Exception handling:** All application function code needs to be reviewed to
  execute proper validation logic and exception handling. Not all cases were
  covered during prototype development.
- **Automated unit, integration and system tests:** In the prototype, some unit
  tests and manual tests were executed but they are by no means comprehensive.
  Automated test scripts are required to achieve higher code coverage and to
  ensure quality and robustness of the solution as it evolves from prototype to
  production.
- **Code repositories and pipelines:** Code repositories allow teams to
  collaborate and CI/CD pipelines allow frequent small improvements to be
  deployed.
- **Amazon Bedrock AgentCore AWS CloudFormation support:** **COMPLETED** - Migrated from custom
  resources to native CloudFormation L1 constructs with Python wrapper
  constructs in `packages/infra/stack/stack_constructs/`.

## Operations

- **Monitoring:** The ability to properly monitor the system in production is
  crucial and the implemented code doesn't provide enough means to do so.
  Consider using both CloudWatch Logs and CloudWatch Metrics to have more
  visibility of all architecture components. For production, consider setting
  the log level to warning to reduce verbosity.

  Specifically, we recommend monitoring the following metrics:
  - **Amazon Bedrock/InputTokens**: Input tokens consumed by the models.
  - **Bedrock/OutputTokens**: Output tokens generated by the models.
  - **AgentCore/Latency**: Response time metrics for chat interactions.
  - **VirtualAssistant/ActiveCalls**: Custom metric tracking concurrent voice
    calls for capacity planning.

- **Auditing:** AWS CloudTrail can be used to implement detective controls.
  CloudTrail records AWS API calls for your account and delivers log files to
  you for auditing.
- **Tracing:** To help with the operations in the production environment, we
  recommend a tracing solution such as AWS X-Ray.
- **VPC Flow Logs:** VPC Flow Logs capture network flow information for a VPC,
  subnet, or network interface and stores it in Amazon CloudWatch Logs and can
  help troubleshoot network issues.

## User Experience Enhancements

- **Audio messages:** Implement support for audio messages sent by
  customers in WhatsApp for example. The message processor could use Amazon
  Polly to transcribe the audio message and pass that to the chat agent.
- **WhatsApp calling:** When a customer initiates a call via WhatsApp, it could
  connect to the voice agent in LiveKit.
- **Cross-modal conversation continuity:** Implement unified session management
  to allow guests to start conversations via voice and continue through chat or
  vice versa without losing context. This requires shared session identifiers
  and extended AgentCore Memory support.

## Architecture Optimizations

- **Voice-Chat Integration:** Explore using the chat assistant as a tool for the
  voice assistant to call instead of using the MCP Server directly. This may
  improve conversation reliability but might increase latency. For best results,
  implement with streaming responses. See
  [Nova Sonic integration examples](https://github.com/awslabs/amazon-bedrock-agentcore-samples/blob/main/03-integrations/nova/nova-sonic/README.md).
- **Production PMS integration:** Replace the simulated Amazon DynamoDB tables with
  direct integration to existing hotel PMS APIs to provide real-time data access
  from production systems.

## AI Model Flexibility

The prototype currently supports Anthropic Claude 3+ and Amazon Nova models.
Extensibility for new models is built into the system. This design allows for
easy adaptation to new AI models or services as they become available or as
customer needs evolve.
